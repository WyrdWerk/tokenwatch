## Context

Worked on the PAYG Inference Calculator project at `~/projects/PAYGO Pricing`. This is a static website that compares pay-as-you-go LLM API pricing across providers. The session covered: initial codebase review, data deduplication (removing LLMGateway mirror and :free entries), deploying to Cloudflare Pages with CI/CD, and a major frontend redesign adding percentage-based token input, org extraction, and an "All models" dropdown option.

## Key Discussion Points

1. **Initial codebase review**: Project is a zero-dependency static site (HTML/CSS/JS) with a Node ESM fetch script that pulls /v1/models from 6 providers, normalizes pricing to $/M tokens, and writes pricing.json. GitHub Actions hourly cron refreshes data.

2. **Overlap handling**: Analyzed 724 model entries — found 15 same-provider duplicates (all OpenRouter :free vs paid pairs), 86 identical-price overlaps between LLMGateway and OpenRouter (LLMGateway is a mirror). User clarified: same model across different providers is fine (that's the value), but same provider+model duplicates must go.

3. **LLMGateway removal**: User directed dropping LLMGateway entirely — it mirrors OpenRouter's catalog at identical prices, adds zero value. Also drop all :free entries ("nobody cares").

4. **Deployment**: GitHub Pages Actions deploy failed with transient "Deployment failed, try again later" on first enablement. Switched to Cloudflare Pages via wrangler CLI. Set up CI/CD with GitHub Actions secrets (CLOUDFLARE_API_TOKEN, CLOUDFLARE_ACCOUNT_ID).

5. **gh CLI auth issue**: GITHUB_TOKEN env var overrides gh CLI keyring token. The env var PAT lacks repo creation scope. Fix: `env -u GITHUB_TOKEN` prefix to use keyring token with repo/workflow scopes.

6. **Schedule change**: User requested daily instead of hourly refresh. Changed cron from `0 * * * *` to `0 0 * * *`.

7. **Frontend redesign**: User provided Google Sheet reference (tiny.cc/payginference) showing: total tokens in millions (1000) + percentage breakdown (2.5% input, 97% cached input, 0.5% output). Replaced 4 absolute token fields with total+percentage model.

8. **Org extraction**: User emphasized OpenRouter is an aggregator, not a provider. Added org field to model records — extracts underlying creator from model ID prefix (e.g., `anthropic/claude-sonnet-5` → org="anthropic"). Cross-references models without org prefixes. Normalizes aliases (deepseek-ai→deepseek, zai-org→z-ai, meta-llama→meta).

9. **All models option**: Added "All models" as first option in model dropdown so users can see all models across all providers without selecting a specific one.

10. **Next phase planning**: User wants further work: (a) fully segregate OpenRouter into underlying providers, (b) aggregate similar model IDs (e.g., Kimi-K2.6 vs kimi-k2.6), (c) replace compare-by dropdown with dual search fields (provider search + model search).

## Next Steps / Follow-up

1. **OpenRouter segregation**: OpenRouter models already have org prefixes (e.g., `mistralai/ministral-3b-2512` → org="mistral"). The frontend should treat org as the actual provider, not OpenRouter. The "Provider" dropdown should list orgs (Anthropic, OpenAI, Mistral, etc.) not aggregators.

2. **Model aggregation**: Current canonicalModelId strips prefix and lowercases, but doesn't handle case variants like `Kimi-K2.6` vs `kimi-k2.6` vs `Kimi-K2.6`. Need more aggressive normalization — possibly strip hyphens/case entirely for matching, or use a fuzzy match.

3. **Dual search UI**: Replace the mode dropdown + model/provider select pattern with two typeahead search inputs. User types in provider field → filters to that provider's models. User types in model field → filters to that model across providers. Both can be used together.
